数据库迁移工具-Migrate

2022-03-24 5 分钟阅读 工具教程数据库

在项目中，因需求的变更常常影响到数据库表结构的设计及数据的更新，导致大量的 sql 脚本难以维护。正因为如此，数据库迁移工具的设计之前，就旨在帮助开发者更合理、有效地管理数据库。

安装migrate

根据migrate/cmd/migrate at master · golang-migrate/migrate (github.com)中的提示选择对应的版本进行安装。

以windows为例，在[Releases · golang-migrate/migrate (github.com)](https://github.com/golang-migrate/migrate)中选择一个版本，找到windows的对应版本，我这里下载migrate.windows-amd64，将里面对应的.exe可执行文件添加到环境变量中。

我将可执行文件放到GOPATH/bin中。

GOPATH的路径可通过go env GOPATH 得到。

1C:\Users\jimyag>migrate -version
24.15.1

安装成功。

使用migrate

1mkdir migration

1migrate create -ext sql -dir migration -seq init_schema_user

将文件拓展名设为sql

要存储的目录为.\migration\,

-seq迁移文件的顺序版本号

init_schema 迁移的名称

执行完命令会生成两个文件

1migration
2├── 000001_init_schema_user.down.sql
3├── 000001_init_schema_user.up.sql

在000001_init_schema_user.down.sql中添加如下内容

多次同步

为了测试 migrations up [N] 执行多次修改的情形，第二次修改我们使用事务为 account表增加 COLUMN，

1D:\Computer\Desktop>migrate create -ext sql -dir migration -seq add_mood_to_user
2D:\Computer\Desktop\migration\000002_add_mood_to_user.up.sql
3D:\Computer\Desktop\migration\000002_add_mood_to_user.down.sql

在000002_add_mood_to_user.up.sql中添加

1BEGIN;
2CREATE TYPE enum_mood AS ENUM (
3	'happy',
4	'sad',
5	'neutral'
6);
7ALTER TABLE accounts ADD COLUMN IF NOT EXISTS mood enum_mood;
8COMMIT;

在000002_add_mood_to_user.down.sql中添加

1BEGIN;
2
3ALTER TABLE accounts DROP COLUMN IF EXISTS mood;
4DROP TYPE enum_mood;
5
6COMMIT;

1D:\Computer\Desktop>migrate create -ext sql -dir migration -seq add_roleid_to_users
2D:\Computer\Desktop\migration\000003_add_roleid_to_users.up.sql
3D:\Computer\Desktop\migration\000003_add_roleid_to_users.down.sql

此时执行，发现错误。

这时候，我们进入数据库中查看schema_migrations表中的数据

migrate的工作流程

up

原有的数据通过产生的up文件向上更改

当我们使用命令migrate up时，产生的up脚本会按照前缀的顺序依次执行

down

新的数据通过down脚本进行还原

当我们使用命令migrate down时，产生的down脚本会按照前缀顺序的逆序进行执行
