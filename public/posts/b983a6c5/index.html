Git删除不小心上传的大文件

2022-04-25 1 分钟阅读 踩坑踩坑Git

在git中不小心上传了大文件，推送到GitHub时被拒绝。

The size of file ‘xxx‘ has exceeded the upper limited size (100 MB) in commit

我们在git中上传文件时不小心上传了压缩之后的public.zip文件，文件大小有108MB，超过了GitHub单个文件记录，被拒绝了。

如何删除呢？

1git filter-branch --force --index-filter "git rm --cached --ignore-unmatch public.zip" --prune-empty --tag-name-filter cat -- --all

其中public.zip就是要删除的大文件名称
